## Two Sum
{   
    Difficulty:         Easy

    Pattern:            HashMap

    Time Complexity:    O(n)

    Space Complexity:   O(n)

    Observation:        Need previous numbers.

    Approach:           Store visited numbers.

    Mistakes:           Initially used nested loops.

    Learning:           HashMap is useful when searching previous elements.
}

Answer for these
{
    How did I recognize this pattern:{
        The problem asks whether two numbers combine to reach a target.
        A brute-force approach checks every pair, which is O(n²). Since we only need to know whether the required complement has already been seen, a hash map provides O(1) average lookup.
        **Recognition clue:**  
        Whenever I need to quickly find whether a previously seen value exists, I should think about using a **Hash Map**.

    }

    Why does this algorithm work:{
        While iterating through the array:
        1. Calculate the complement:
            complement = target - nums[i]
        2. Check if the complement already exists in the hash map.
        3. If it exists, we've found the required pair.
        4. Otherwise, store the current element and continue.
        Since every element is processed once and hash map lookups are O(1) on average, the overall complexity is O(n).
    }

    What mistake did I make the first time:{
        - My first idea was to compare every pair using two loops.
        - That works, but its complexity is O(n²), which is inefficient.
        - Another common mistake is inserting the current number into the hash map **before** checking for its complement. For problems with duplicate values, that can lead to incorrect behavior. It's safer to check first, then insert.

    }
    
    When should I use this technique again:{
        Think of a **Hash Map** whenever:

        - I need fast lookup of previously visited elements.
        - The problem asks for a pair, frequency, or complement.
        - I want to reduce nested loops to a single pass.
        - Lookup time is more important than maintaining order.
    }
}
